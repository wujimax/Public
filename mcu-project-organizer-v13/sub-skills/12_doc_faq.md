# 子Skill 12: 生成 11_常见问题清单.md

> **职责**: 预生成20个项目常见问题+答案，主要供 mcu-qa-assistant 消费，同时对新人也有价值。
> **输入**: 已生成的知识库文档（02/03/05/07）+ `_v10_snapshot/sources/`
> **输出**: `11_常见问题清单.md`

---

## 必须遵守的共享规则

开始前先读取:
- `shared/iron_rules.md` — 铁律规则（特别注意铁律8"直接读源码"和铁律9"补厚不覆盖"）
- `shared/format_spec.md` — 排版规范

## 执行前：判断运行模式

### 模式判断
1. 检查目标文档是否存在（`11_常见问题清单.md`）
2. 存在 → **补厚模式**：读取现有文档，只补充缺失内容
3. 不存在 → **全新生成模式**：从零开始生成

### 补厚模式操作规则
- 读取现有问题列表，统计各类别已有数量
- 只补充缺失的类别/数量（如"调试类只有2个，需要补2个"）
- 不修改已有的问题和答案
- 在文档末尾追加 `## 补充问题（第X批）`
- 执行完成后输出操作摘要

### 数据读取规则（全新/补厚均适用）
1. 读取已生成的知识库文档作为答案素材：
   - `02_硬件配置.md`（引脚/外设问题的答案）
   - `03_系统架构.md`（架构/调度问题的答案）
   - `05_通信协议.md`（协议问题的答案）
   - `07_已知问题与建议.md`（调试问题的答案）
2. 需要补充细节时，直接读 `_v10_snapshot/sources/` 源码
3. 数值参数交叉验证 `06_关键参数表.md`

## 设计原则

- 问题来自代码分析，不是凭空想象
- 每个答案必须有代码位置（文件:行号）
- 问题分5类，每类4个
- 答案必须直接可用，不能是"请看文档"这种废话
- 如果答案是AI推断，必须标注

---

## 问题分类和数量

| 类别 | 数量 | 问题类型 | 示例 |
|------|------|---------|------|
| **功能类** | 4个 | 这个项目是什么/能做什么 | "这个项目的主要功能是什么？" |
| **配置类** | 4个 | 在哪里改/怎么改 | "串口波特率在哪里修改？" |
| **调用类** | 4个 | 怎么用/如何调用 | "如何发送一条MQTT消息？" |
| **调试类** | 4个 | 出问题了怎么排查 | "NBIoT连接不上，怎么排查？" |
| **扩展类** | 4个 | 如何新增/改功能 | "如何新增一个传感器类型？" |

**总计：20个问题**

## 每条Q&A的格式（必须严格遵守）

```markdown
---
**Q{序号}** [{类别}] {问题文本}

**答案**：
{直接给出答案，2-5句话，结论先行}

**关键代码位置**：
- `{文件名}:{行号}` — {一句话说明这里有什么}
- `{文件名}:{行号}` — {一句话说明这里有什么}

**延伸问题**：
- {如果想进一步了解，可能会问的下一个问题}
- {另一个相关问题}

**置信度**：{高 / 中 / 低}
{如果是中/低，加：⚠️ 部分内容为AI推断，建议核实}

---
```

## 实际示例

```markdown
---
**Q01** [功能类] 这个项目是做什么用的？

**答案**：
这是一个家用安防报警主机，能接收433MHz无线门磁/遥控器的信号，
判断是否触发报警，并通过NBIoT网络把报警信息发送到OneNet物联网平台。
用户可以通过手机App或遥控器进行布防/撤防。

**关键代码位置**：
- `app.c:1` — 主业务逻辑，布防/撤防/报警状态机
- `hal_NBIot.c:1` — NBIoT通信，连接OneNet平台
- `hal_rfd.c:1` — 433MHz无线信号接收和解码

**延伸问题**：
- 系统支持几个传感器？（Q05）
- 布防之后什么情况会触发报警？（Q03）

**置信度**：高

---

**Q09** [调试类] NBIoT连接不上OneNet，从哪里开始排查？

**答案**：
按顺序检查：① SIM卡是否就绪（看 `+CPIN: READY`）→ ② 网络是否附着（`+CGATT: 1`）
→ ③ MQTT服务器地址是否正确（`183.230.40.96:1883`）→ ④ ProductID和Token是否匹配。
调试串口会打印每步AT指令的响应，波特率9600。

**关键代码位置**：
- `hal_NBIot.c:1073` — AT响应解析，每个响应对应一个枚举值
- `hal_NBIot.c:241` — 初始化流程，按此顺序追查哪步卡住了
- `hal_NBIot.c:18` — ProductID和OneNet服务器地址硬编码在此

**延伸问题**：
- 连接成功后数据发送失败怎么排查？（Q10）
- 如何修改连接的OneNet账号？（Q07）

**置信度**：高

---
```

## 生成Q&A时的AI自检规则

生成每条Q&A前，必须确认：
1. 答案中提到的函数名，在源码中真实存在
2. 文件:行号引用是真实的，不是估算的
3. 答案中的数值（波特率/IP地址/超时时间等）来自 `06_关键参数表.md` 或源码
4. 不生成"超出代码范围"的问题（如"如何连接WiFi" — 这个项目没有WiFi）

---

## 输出要求

- 共20个问题，5类各4个
- 每条Q&A必须有"关键代码位置"字段（至少1个文件:行号）
- AI推断的答案必须有⚠️标注
- 延伸问题应尽量引用现有的其他问题编号，构成问题网络
- 文档开头提供分类统计和问题索引
