# 子Skill 1: 生成 00_阅读指南.md + 01_项目介绍.md

> **职责**: 读取 `_analysis/` 中的分析结果，生成阅读指南和项目介绍两个文档。
> **输入**: `_analysis/project_overview.md`, `_analysis/data_structures.md`, `_analysis/issues_and_risks.md`, `_analysis/peripheral_config.md`
> **输出**: `00_阅读指南.md`, `01_项目介绍.md`

---

## 必须遵守的共享规则

开始前先读取:
- `shared/iron_rules.md` — 铁律规则
- `shared/format_spec.md` — 排版规范

---

## 执行前：判断运行模式

### 模式判断
1. 检查目标文档是否存在（`00_阅读指南.md` 和 `01_项目介绍.md`）
2. 存在 → **补厚模式**：读取现有文档，只补充缺失内容
3. 不存在 → **全新生成模式**：从零开始生成

### 补厚模式操作规则
- 读取现有文档，记录"已有哪些章节/内容"
- 与本子skill的"应有内容"对比，找出缺失项
- 只追加缺失内容，不修改已有内容
- 在追加内容前加注释：`<!-- 补充于 [日期] -->`（可选）
- 执行完成后输出操作摘要

### 数据读取规则（全新/补厚均适用）
1. 先读 `_analysis/project_overview.md` 等 → 了解有哪些源文件需要读
2. 直接打开 `_v10_snapshot/sources/` 中的源文件
3. 从源码提取完整数据（数值必须有 源文件:行号 来源）
4. `_analysis/` 只作为导航，文档中所有数据来自源码

## 文档 1: `00_阅读指南.md`

### 必须包含: 项目速览封面表

```markdown
# 项目数字资产档案
## [项目名称] [版本号]

### 项目概况速览
| 指标 | 数据 |
|---|---|
| 芯片型号 | [精确到系列，如 STM32F105 CL] |
| 代码规模 | X个业务源文件, ~Y行有效代码 |
| 外设使用 | [简要列出，如 4xUSART + SPI + I2C + ADC + 3xTIM] |
| 通信方式 | [如 4G/WIFI/Lora三网 + MQTT] |
| 传感器支持 | [如 20防区, EV1527 + Lora] |
| 发现问题 | X个BUG + Y处注释错误 + Z项风险 |
| AI分析覆盖率 | 100%业务源文件 |

> 本档案由AI自动分析生成，所有结论标注源文件和行号，可直接追溯验证。
```

**数据来源**:
- 芯片型号 → `_analysis/project_overview.md`
- 代码规模 → `_analysis/project_overview.md`
- 外设使用 → `_analysis/peripheral_config.md`
- 发现问题 → `_analysis/issues_and_risks.md`

### 必须包含: 文档导航

列出9个文档的名称和简要说明。

## 文档 2: `01_项目介绍.md`

### 必须包含的章节

1. **项目名称和用途** — 这是一个什么产品？
2. **功能列表** — 具体能做什么？逐条列出
3. **目标用户/应用场景**
4. **固件版本信息** — 如果代码中有版本号
5. **通信方式概述** — 用了哪些通信手段？
6. **与外部系统的对接关系**

### 必须包含: 代码统计表

| 指标 | 数据 |
|------|------|
| 业务源文件数 | X |
| 第三方库文件数 | Y |
| 有效代码行数 | ~Z |
| 函数数量 | W |
| 全局变量数 | V |

**数据来源**: `_analysis/project_overview.md`

### 必须包含: 源码目录结构树

带文件大小和职责标注的目录树。

**数据来源**: `_analysis/project_overview.md`

### 必须包含: 核心数据结构定义

列出项目中关键的 struct/enum 定义及其字段含义。

**数据来源**: `_analysis/data_structures.md`

- 如果 `_analysis/` 中缺少某项数据，回查源码补充（读 `_v10_snapshot/`）
