## 执行前：判断运行模式

### 模式判断
1. 检查目标文档是否存在（`08_数据流与控制流.md`）
2. 存在 → **补厚模式**：读取现有文档，只补充缺失内容
3. 不存在 → **全新生成模式**：从零开始生成

### 补厚模式操作规则
- 读取现有文档，记录"已有哪些章节/内容"
- 与本子skill的"应有内容"对比，找出缺失项
- 只追加缺失内容，不修改已有内容
- 在追加内容前加注释：`<!-- 补充于 [日期] -->`（可选）
- 执行完成后输出操作摘要

### 数据读取规则（全新/补厚均适用）
1. 先读 `_analysis/data_flow.md` → 了解有哪些源文件需要读
2. 直接打开 `_v10_snapshot/sources/` 中的源文件
3. 从源码提取完整数据（数值必须有 源文件:行号 来源）
4. `_analysis/` 只作为导航，文档中所有数据来自源码

### 2. 核心数据链路追踪

追踪 2-3条核心业务场景的端到端数据链路，从物理输入到最终输出。

**必须追踪的场景（按项目类型选择）:**
- **报警类项目**: 探测器触发 → 无线接收 → 解码 → 匹配 → 模式切换 → 本地报警 → 云端上报 → APP推送
- **通信类项目**: 外部数据输入 → UART接收 → 协议解析 → 业务处理 → 响应构造 → 发送
- **控制类项目**: 传感器采样 → ADC转换 → 滤波算法 → PID计算 → PWM输出 → 执行器动作

#### 每条链路格式

```markdown
#### 链路1: [场景名称]

**触发条件**: [什么情况触发这条链路]

##### 数据流节点

1. **[节点名称]** (xxx.c:行号)
   - 输入: [数据格式/类型]
   - 处理: [核心逻辑一句话]
   - 输出: [传递方式] → 下一节点
   - 关键代码: `[关键函数调用或变量赋值]`

2. **[节点名称]** (xxx.c:行号)
   - 输入: ...
   - 处理: ...
   - 输出: ...

##### 链路 Mermaid 图

​```mermaid
flowchart TD
    A["物理输入\n(xxx.c:行号)"] --> B["中断接收\n(xxx.c:行号)"]
    B --> C["协议解析\n(xxx.c:行号)"]
    C --> D["业务处理\n(xxx.c:行号)"]
    D --> E["输出响应\n(xxx.c:行号)"]
​```
```
