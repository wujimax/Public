# 可信度/置信度统一标准

> 本文件被所有子skill引用，统一整个知识库的可信度表述。
> 禁止在文档中自创置信度表示方式。

---

## 统一术语

全库只使用以下两个术语，含义相同，仅因使用场景不同而选用：

| 场景 | 使用术语 | 示例 |
|------|---------|------|
| Q&A 条目（11_FAQ） | `置信度` | `**置信度**：高` |
| 经验库条目（12_历史经验库） | `可信度` | `**可信度**：★★★` |
| 验收包 Part 1（15_验收包） | `可信度` | `**可信度**：中` |
| 其他文档的标注 | `[可信度：高/中/低]` | `72MHz [可信度：高]` |

---

## 三级评级定义

### 高（★★★）
- 数值/结论直接来自源码赋值语句（铁律0：代码即事实）
- 有明确的 `文件:行号` 作为证据
- 经过至少一次交叉验证（如参数表和源码一致）

### 中（★★）
- 来自注释、宏定义（非直接赋值）
- 来自 `_analysis/` 导航索引，未回源码验证
- 来自经验库的转述记录
- 来自 AI 从代码上下文推断（需加 `[AI推断]` 标注）

### 低（★）
- 来自代码注释（可能与实现不一致）
- 口述未验证的经验
- 基于芯片手册默认值的推断（未在项目代码中找到对应配置）
- 来源不明

## 显示格式规范

### Q&A 条目（11_FAQ）中

```markdown
**置信度**：高
```
或
```markdown
**置信度**：中
⚠️ 部分内容为 AI 推断，建议核实
```

### 经验库条目（12_历史经验库）中

```markdown
**可信度**：★★★
```
或
```markdown
**可信度**：★★
```

### 普通文档数值标注中

直接在数值后加 `[可信度：高/中/低]`：
```
波特率：115200 [可信度：高]（来源：usart.c:48）
射频频率：433MHz [可信度：低][需硬件资料确认]
```

---

## 评级降级触发规则

出现以下任一情况，可信度自动降级：

| 触发条件 | 降级幅度 |
|---------|---------|
| 来源未标注（无文件:行号） | 降一级 |
| 仅来自注释，未验证 | 降一级 |
| 经验库条目来源为"口述" | 降一级 |
| 信息来自 `_analysis/` 未回源码 | 降一级 |
| AI 推断（加 `[AI推断]` 后）| 不降整体等级，但该字段单独标注 |

---

## 禁止的写法

| 禁止 | 应改为 |
|------|-------|
| `可信度：很高` | `可信度：高` |
| `置信度：★★` | `置信度：中` |
| `确认度：✅` | `置信度：高` 或直接写确认度标识（见format_spec.md） |
| `准确性：未知` | `可信度：低` |
| 不写可信度 | 必须写，哪怕是"高" |
