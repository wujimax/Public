# 排版规范 (文档生成子skill引用)

> 所有文档生成子skill必须遵守以下排版规范，确保跨项目一致性。

---

## 标题层级

- `#` 文档标题 (每个文件只有一个)
- `##` 大章节
- `###` 模块/子章节
- `####` 子话题（如"为什么XXX"、"对外API"）

## 表格规范

- 所有外设表必须有"确认度"列 (✅代码确认 / ⚠️需硬件确认)
- 所有参数表必须有"来源"列 (文件:行号)
- 命令/枚举表必须有"数值"列
- 表格列对齐，同类表格在不同项目中保持一致格式

## Mermaid图使用要求

- 状态机: `stateDiagram-v2`
- 流程图: `flowchart TD`
- 时序图: `sequenceDiagram`
- 依赖关系: `graph LR`
- 同时保留文字描述（图+文字互补）

## 分隔符

- 同级模块之间用 `---` 分隔
- 不要在 `##` 标题后紧跟 `---`

## 确认度标识

- ✅代码确认: 有源文件:行号直接证据
- ⚠️需硬件确认: 代码中无法确定，需要查阅原理图/datasheet
- 🔍推断: 从代码上下文推断，但无直接证据

## 可改性标识 (用于参数表)

- ⚙️可改: 可根据需求修改
- ⚠️改则不兼容: 修改后会导致与现有系统不兼容
- 🔒不建议改: 修改可能导致系统异常

---

## 来源字段统一格式（L4修复）

全库所有"来源"字段必须使用以下统一格式，不得自创：

```
sources/文件名.c:行号
```

示例：
- ✅ `sources/hal_usart.c:48`
- ✅ `sources/app.c:125`
- ❌ `hal_usart.c:48`（缺少路径前缀）
- ❌ `hal_usart.c:USART_Init()`（用函数名代替行号）
- ❌ `hal_usart.c`（无行号）
- ❌ `相对路径/hal_usart.c:48`（路径格式错误）

**特殊情况**：
- 头文件：`headers/stm32f10x_usart.h:32`
- 不确定行号时：`sources/hal_usart.c:[约48行]`，不得省略文件名

## Mermaid图类型速查（L1修复）

| 用途 | 图类型 | 示例声明 |
|------|-------|---------|
| 状态机/状态转换 | `stateDiagram-v2` | 报警状态、连接状态 |
| 启动/执行流程 | `flowchart TD` | 上电流程、任务调度 |
| 时钟树/层级关系 | `flowchart LR` | 时钟分频树、软件分层 |
| 模块依赖/调用关系 | `graph LR` | 模块间 include 依赖 |
| 时序/通信交互 | `sequenceDiagram` | 串口握手、MQTT连接 |
| 数据流/管道 | `flowchart TD` | 数据接收处理链路 |

**禁止**：不可因为"这个类型好看"而选错类型；不可用 `graph TD` 代替 `stateDiagram-v2` 画状态机。

## 代码片段展示规范（L3修复）

| 场景 | 最大展示行数 | 处理超出部分 |
|------|-----------|-----------|
| BUG证据（07_已知问题）| 15行 | 只展示问题行 ±5 行上下文 |
| 函数卡片（10_代码语义化）| 10行 | 只展示函数签名+关键逻辑行 |
| 协议帧格式（05_通信协议）| 20行 | 完整展示（帧定义天然短） |
| 数据结构（举例说明）| 10行 | 只展示关键字段 |

超出限制时，用注释说明截断位置：
```c
// ... 省略 {N} 行非关键逻辑，见 sources/xxx.c:{起始行}-{结束行}
```

## 不确定性标注统一写法（L8修复）

全库只使用以下三种标注，不可混用：

| 标注 | 含义 | 使用场景 |
|------|------|---------|
| `[需硬件资料确认]` | 代码中无法确定，需要原理图/datasheet | 射频频率、外部晶振值、PCB走线 |
| `[AI推断]` | 从上下文推断，非直接代码证据 | 设计意图、参数选择原因 |
| `[需查阅原理图]` | 特指需要原理图才能确认 | 引脚连接、外部器件型号 |

**禁止**：`[需查头文件]`（头文件在snapshot里必须去读）、`[未知]`、`[不确定]`、`[待确认]`
