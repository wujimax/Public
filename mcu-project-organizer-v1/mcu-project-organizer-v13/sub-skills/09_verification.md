# 子Skill 9: 验证与修正

> **职责**: 所有12+1个文档生成完毕后，执行全面验证，输出修正清单。
> **输入**: 12+1个输出文档 + `_analysis/` 中间产物 + 源码快照
> **输出**: 修正操作（直接修复文档中的问题） + 验证报告

---

## 必须遵守的共享规则

开始前先读取:
- `shared/quality_checklist.md` — 完整的质量检查清单
- `shared/iron_rules.md` — 铁律规则

---

## 验证流程

### 步骤 1: 运行自动化验证脚本

```
python scripts/verify_output.py <知识库目录> <_v10_snapshot/sources目录>
```

检查项:
- 文档命名(12+1个)
- GPIO引脚覆盖率
- 外设枚举覆盖率
- 跨文档一致性
- 必备内容
- 源文件覆盖率
- 宏引用实际使用验证

**失败项记录到修正清单。**

### 步骤 2: 完整性检查

#### A. 未解析标识符扫描

在所有输出文档中搜索以下关键词:
- `需查头文件` — snapshot中存在的头文件不应有此标注
- `_PIN`、`_PORT` — 可能是未解析的宏名
- `_MAX`、`_SIZE`、`_SUM` — 可能是未解析的常量宏
- `STM32F10x`、`STM32F4xx` — 芯片型号模糊写法

对每个搜到的未解析标识符:
1. 去 `_v10_snapshot/headers/` 中查找定义
2. 替换为实际值
3. 只有 snapshot 中确实不存在的才可标注 `[需硬件资料确认]`

#### B. 芯片型号模糊检测

搜索所有 `STM32F10x`、`STM32F4xx` 等模糊写法。
回查 `_analysis/project_overview.md` 获取精确型号并修正。

#### C. 源文件覆盖率

逐个检查 `project_context.json` 的 `business_sources` 列表中每个文件是否在 `04_功能模块.md` 中被提及。遗漏的文件需要回查源码补充到 04 文档中。

#### D. Mermaid图存在性

确认:
- [ ] 02 有时钟树图
- [ ] 03 有启动流程图
- [ ] 03 有分层架构总览图
- [ ] 04 有模块依赖图
- [ ] 05 每个通信协议/AT状态机有状态图
- [ ] 09 有分层架构图
- [ ] SUMMARY 有架构图（从09复制）

缺失的图需要读取 `_analysis/` 中的数据补充。

#### E. 问题分级完整

确认 07 中:
- [ ] 每个问题都有 P0/P1/P2/P3 标签
- [ ] 每个问题都有文件名+行号

### 步骤 3: 正确性回查验证

从输出文档中随机抽取关键断言，**回到源码**验证:

1. **时钟树验证**: 
   - 读 02_硬件配置中的 SYSCLK/APB1/APB2 值
   - 回查 `_v10_snapshot/` 中 system_stm32f10x.c 的宏和分频寄存器配置
   - 重新计算确认

2. **引脚映射全覆盖**:
   - 外设总表中每一个引脚必须标注源文件:行号
   - 没有源文件行号的引脚条目视为不合格

3. **通信参数验证**:
   - 波特率 → 回查 USART_Init()
   - 服务器地址 → 回查 AT 指令字符串
   - 命令字数值 → 回查 enum 定义

4. **状态机完整性**:
   - 文档中画的状态机 → 回查代码 switch/case
   - 确认没有遗漏 case 分支
   - 确认转换条件正确

5. **BUG验证**:
   - 07 文档中每个 P0 BUG → 回到源码对应行号
   - 确认代码确实存在且确实是 BUG

6. **跨文档一致性**:
   - 02 中的波特率 = 06 关键参数表中的波特率?
   - 02 中的引脚 = 04 中提到的引脚?
   - 不一致则回查源码修正

7. **外设列表一致性**:
   - 01/00 文档提到的每个外设(OLED/UART/SPI等)
   - 在 02_硬件配置中必须有引脚+参数
   - 在 04_功能模块中必须有模块描述
   - 三处不一致则回查源码修正

8. **外设接口精确性**:
   - 禁止"I2C/SPI"这种模糊表述
   - 必须确定是I2C还是SPI
   - 是硬件外设还是软件模拟
   - 有代码证据支撑

9. **Q&A代码位置验证**:
   - 11_常见问题清单中的每个文件:行号 → 回查源码确认存在
   - 答案中提到的函数名必须在源码中真实存在
   - 数值参数与 `06_关键参数表.md` 一致

10. **函数卡片覆盖率验证**:
   - 对照 `call_graph_hint.json` 检查优先级1文件的公开函数是否都有1张卡片
   - 覆盖率 < 90% 视为不合格

### 步骤 4: 修正

对步骤1-3中发现的所有问题:
1. 直接修改对应的输出文档
2. 记录修正内容

### 步骤 5: 生成验证报告

在知识库目录下生成 `_verify_report.txt`:

```
验证时间: [时间戳]
验证结论: [✅全部通过 / ⚠️有警告 / ❌有失败]

--- 自动化脚本结果 ---
[verify_output.py 的输出]

--- 手动验证结果 ---
[各项检查的结果]

--- 修正记录 ---
[修正了哪些问题]
```
